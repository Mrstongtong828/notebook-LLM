## GitHub 项目推荐

今日主题：Agent 凭证安全与本地工作流防泄漏。原因：近期本地笔记中出现过明文 API key，且昨天已经确认 Agent 工具链的真实风险常常来自凭证和第三方包。

### 1. gitleaks/gitleaks

- 一句话判断：用于扫描 Git 仓库、文件和提交历史中的密钥泄漏。
- 星标/热度或受关注原因：GitHub API 核验约 27574 stars，2026-06-07 仍有更新。
- 原始链接：https://github.com/gitleaks/gitleaks
- 适合我的原因：适合给 Obsidian Vault、代码项目和自动化脚本做本地密钥扫描。用途：项目复用、安全习惯。

### 2. trufflesecurity/trufflehog

- 一句话判断：查找、验证和分析泄漏凭证，能覆盖 Git、文件系统、CI、云服务等场景。
- 星标/热度或受关注原因：GitHub API 核验约 26667 stars，2026-06-07 仍有更新。
- 原始链接：https://github.com/trufflesecurity/trufflehog
- 适合我的原因：比简单正则扫描更偏“验证凭证是否真实有效”，适合后续做一次本地仓库和笔记库安全体检。用途：学习、项目复用。

### 3. Infisical/infisical

- 一句话判断：开源 secrets、证书和 privileged access 管理平台。
- 星标/热度或受关注原因：GitHub API 核验约 27259 stars，2026-06-07 仍有更新。
- 原始链接：https://github.com/Infisical/infisical
- 适合我的原因：适合学习如何把 API key 从笔记和配置文件中迁移到统一管理工具。用途：学习、专业方向。

### 4. GitGuardian/ggshield

- 一句话判断：GitGuardian 的命令行工具，可作为 pre-commit hook、GitHub Action 或本地 CLI 检测硬编码密钥。
- 星标/热度或受关注原因：GitHub API 核验约 1961 stars，2026-06-05 仍有更新。
- 原始链接：https://github.com/GitGuardian/ggshield
- 适合我的原因：适合接到小型项目开发流程里，先从 pre-commit 防止误提交密钥开始。用途：项目复用、开发规范。
