## GitHub 项目推荐

今日主题：终端 Coding Agent、浏览器 MCP 和可审查执行链路。原因：今日 OpenAI、GitHub 和 NVIDIA 的动态都指向“AI 不只回答，而是在受控环境里执行、审查、留下记录”。

### 1. openai/codex

- 一句话判断：OpenAI 的轻量终端 Coding Agent，适合学习 Codex 式本地开发、工具调用和代码库任务执行。
- 星标/热度或受关注原因：`gh` 核验约 90257 stars，2026-06-11 仍有推送更新。
- 原始链接：https://github.com/openai/codex
- 适合我的原因：适合学习 Agent 如何在终端中读代码、改文件、跑验证；也可直接作为 Horizon 修复和竞赛项目整理的主工具链。

### 2. google-gemini/gemini-cli

- 一句话判断：Google 开源的 Gemini 终端 AI Agent，可作为 Codex/Claude Code 的横向对照样本。
- 星标/热度或受关注原因：`gh` 核验约 105148 stars，2026-06-10 仍有推送更新。
- 原始链接：https://github.com/google-gemini/gemini-cli
- 适合我的原因：适合对比不同 CLI Agent 的权限模型、工具接口、上下文组织和开发体验，帮助你形成自己的 Agent 工程判断。

### 3. microsoft/playwright-mcp

- 一句话判断：把 Playwright 浏览器自动化包装成 MCP server，让 Agent 可以更标准地操作网页和做端到端验证。

- 原始链接：https://github.com/microsoft/playwright-mcp
- 适合我的原因：可用于小程序后台/网页项目验收、Horizon 信息源检查、竞赛项目演示流程录制；比单纯聊天更接近真实可验证执行。
