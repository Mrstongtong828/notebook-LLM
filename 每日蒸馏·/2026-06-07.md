## GitHub 项目推荐

今日主题：Agent/RAG 工作流的可观测性、评估与调试，服务 `Horizon 信息雷达` 和 `AI Agent 执行环境`。

### 1. langfuse/langfuse

- 一句话判断：开源 LLM 工程平台，覆盖 tracing、metrics、evals、prompt 管理、数据集和 OpenTelemetry 集成。
- 星标/热度或受关注原因：GitHub API 核验约 28596 stars，2026-06-07 仍有更新。
- 原始链接：https://github.com/langfuse/langfuse
- 适合我的原因：适合学习如何给 Horizon/Codex 类工作流增加调用链、输入输出记录和评估。用途：学习、项目复用、专业方向。

### 2. comet-ml/opik

- 一句话判断：面向 LLM 应用、RAG 和 agentic workflow 的调试、评估与监控平台。
- 星标/热度或受关注原因：GitHub API 核验约 19436 stars，2026-06-07 仍有更新。
- 原始链接：https://github.com/comet-ml/opik
- 适合我的原因：它的“trace + eval + dashboard”方向适合借鉴到每日蒸馏：每次跑失败时记录失败阶段、耗时、输入源和降级路径。用途：学习、项目复用。

### 3. Arize-ai/phoenix

- 一句话判断：AI observability 与 evaluation 工具，可用于排查 LLM/RAG/Agent 的质量和链路问题。
- 星标/热度或受关注原因：GitHub API 核验约 10008 stars，2026-06-06 仍有更新。
- 原始链接：https://github.com/Arize-ai/phoenix
- 适合我的原因：适合用来理解“如何判断一个 Agent 工作流是否真的变好”，而不是只看最终回答是否好看。用途：学习、专业方向。
