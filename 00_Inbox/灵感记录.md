## 一、BRAT — Beta Reviewer's Auto-update Tool (`obsidian42-brat`)

**用途**：安装和自动更新**未上架社区商店**的插件/主题（直接从 GitHub 下载）。

### 适用场景

- 插件的 GitHub 仓库有 Release，但还没进 Obsidian 社区商店
- 你想抢先试用 beta 版插件
- 开发者给你一个 GitHub 链接让你帮忙测试
### 用法

1. **打开命令面板**（`Ctrl+P`）→ 搜索 `BRAT: Add a beta plugin for testing`
2. **输入 GitHub 仓库地址**，例如：
   - `https://github.com/TfTHacker/obsidian42-brat`（BRAT 自己）
   - `https://github.com/mara-li/obsidian-bartender`
3. **点击「Add Plugin」** → 自动下载安装
4. 去「设置 → 第三方插件」里启用它
5. **更新**：运行 `BRAT: Check for updates to all beta plugins and UPDATE`

### 用在你这里

你的 Claudian 插件（`realclaudian`）是通过 `main.js` 直接放进来的，目前不在社区商城。以后如果作者通过 GitHub Release 发布更新版本，你可以用 BRAT 来管理自动更新，不用手动下载替换文件。

## 二、Git 插件 (`obsidian-git`)

**用途**：用 Git 给你的 Obsidian Vault 做**版本控制**和**自动备份**。

### 核心功能

| 功能 | 说明 |
|------|------|
| **自动备份** | 每隔 N 分钟自动 commit + push 到 GitHub/Gitee |
| **版本历史** | 可以回溯到任意时间点的内容 |
| **拉取更新** | 多设备间同步（配合 GitHub） |
| **差异对比** | 查看文件修改了哪些内容 |

### 设置步骤

1. **先保证你的电脑上装了 Git**（`git --version` 确认）
2. **在 Obsidian 设置 → 第三方插件 → 启用 obsidian-git**
3. **设置 → obsidian-git**：
   - `Vault backup interval (minutes)` → 设成 `30` 或 `60`（自动备份间隔）
   - `Auto commit message` → 如 `"vault backup {{date}}"`
   - `Pull on backup` → 多设备同步时开启
4. **首次手动备份**：`Ctrl+P` → `Obsidian Git: Create backup`
5. **它会自动创建一个 `.git` 仓库并开始追踪所有变**

### 结合你的 Vault

```bash
# 如果你想关联到远程仓库（多设备同步）：
# 先在 GitHub 创建一个私有仓库，然后在 vault 目录执行：
git remote add origin https://github.com/你的用户名/你的仓库.git
git push -u origin main
```

之后 Git 插件自动 push/pull，你的笔记就在云端有备份了。

## Terminal

**在 Obsidian 里直接运行终端/命令行**，不用切窗口。

- 集成终端（面板内）和外部终端（弹出独立窗口）
- 你目前的配置：`win32IntegratedDefault` → 默认用 `cmd.exe`

**用法**：
| 操作 | 方式 |
|------|------|
| 打开终端 | `Ctrl+P` → `Terminal: New` |
| 新建分栏 | 默认在当前面板右侧横向分栏 |
| 发送命令 | 直接输入 + `Shift+Enter`（发送不换行） |
| 支持 Shell | cmd.exe（已配好），也可改配置成 PowerShell、Git Bash |

**快捷键**：
- `Shift+Enter` — 发送命令
- `PgUp/PgDn` — 滚动
- `Home/End` — 顶部/底部

## Excalidraw

- **定位**：在 Obsidian 里画手绘风格图表，与双向链接打通
- **文件格式**：`.md` + 内嵌 SVG/JSON → 纯文本、可搜索、可 Git 追踪
- **核心用法**：
  - `![[绘图.excalidraw]]` 嵌入笔记，支持 `![[绘图.excalidraw|300|left]]` 调尺寸/对齐
  - 画布中写 `[[笔记名]]` 创建链接形状 → 点击跳转，Graph View 可见
  - `Shift + 拖拽` 笔记到画布 → 渲染为实时更新的卡片
  - Frame 框选区域 → 右键复制区块引用 → `![[绘图.excalidraw#^area]]` 局部嵌入
- **快捷键**：`1-9` 切工具，`R` 矩形、`O` 椭圆、`T` 文字、`A` 箭头、`P` 铅笔
- **建议设置**：
  - 默认文件夹 → `Attachments/Excalidraw/`
  - 文件名模板 → `Drawing YYYY-MM-DD HH.mm.ss`
- **适用场景**：概念图/架构图/头脑风暴/流程图/MOC 看板
- **慎用场景**：纯数据表格/大而全的单张画布（影响性能）

## Local Images Plus

- **定位**：把笔记中的网络图片自动下载到本地，防止失效
- **核心用法**：
  - **自动模式**：开启 `Automatic processing`，粘贴即下载
  - **手动模式**：粘贴后执行 `Localize attachments for current note`
  - 支持网页 `<img>`、Word/Office 嵌入图、Base64、Markdown `![](url)` 图片
- **实用命令**：
  - `Localize attachments for all notes` — 批量处理整个 vault
  - `Remove all orphaned attachments` — 清理无人引用的附件
- **建议配置**：
  - 附件路径 → `{{noteFolder}}/{{noteName}}_attachments/`
  - 开启 MD5 去重
- **⚠️ 注意**：批量处理前先备份 vault；与 Paste Image Rename / Pretty BibTex 冲突
