## 1. 架构概述
[cite_start]视频工厂的核心目标是作为“内容发动机”，为市场部每日输出短视频脚本、图文素材等 [cite: 15]。  
[cite_start]本工作流实现“短视频生产流水线” [cite: 44]，整体串联逻辑如下：

`开始节点` ➡️ `插件节点(热点监测)` ➡️ `大模型节点1(匹配选题)` ➡️ `大模型节点2(生成脚本)` ➡️ `代码节点(格式化)` ➡️ `输出节点(推送审核)`

---

## 2. 节点详细配置指南
### 节点 1：【开始】 (Start Node)
+ **功能描述：** 接收用户（市场部同学）的基础触发指令。
+ **输入参数配置：**
    - [cite_start]`content_type` (String): 内容类型（例如：解题技巧短视频、学情案例展示、热点借势、教师IP口播） [cite: 42]
    - `target_audience` (String): 目标受众（例如：初三家长、高一学生）
    - [cite_start]`video_duration` (String): 视频时长预期（例如：15秒、30秒、60秒） [cite: 40]

### [cite_start]节点 2：【热点监测】 (Plugin Node) [cite: 45]
+ [cite_start]**功能描述：** 自动抓取抖音/视频号教育类热点话题 [cite: 45]。
+ **实现路径（本地协同）：**
    - **开发方案：** 扣子原生插件可能无法精准抓取佛山本地教育热点。建议小组成员使用本地环境编写 Python 爬虫脚本抓取热榜榜单，并将其封装为自定义 API 供 Coze 调用。
    - **⚠️**** 本地调试规范 (opencode执行)：** 在开发和维护用于此节点的本地服务代码时，请务必保持项目目录的整洁。如果挂载在现有的自动化工程下，**执行前务必检查并删掉内外层无关的冗余测试目录，仅留取当前我们要用的那个本地版本**。这样能确保在使用 `opencode` 执行和调试时，有效避免因环境混乱导致的手误执行报错。

### [cite_start]节点 3：【选题匹配】 (LLM Node 1) [cite: 46]
+ [cite_start]**功能描述：** 结合开卓课程体系和抓取到的热点，匹配并输出 3 个备选选题大纲 [cite: 46]。
+ [cite_start]**模型选择：** 豆包 [cite: 38]
+ [cite_start]**开发流：** * 在此节点填写的 System Prompt，请交给 **Claude** 来起草。将开卓的核心优势（如：全学段覆盖、主打数理化等 [cite: 6]）喂给 Claude，让其生成结构化的提示词。
+ **输入流：** 节点 1 (用户参数) + 节点 2 (热榜数据)

### [cite_start]节点 4：【脚本生成】 (LLM Node 2) [cite: 47]
+ [cite_start]**功能描述：** 将选定的大纲扩写为完整的输出。包含：文案、分镜、字幕 [cite: 40, 47]。
+ **开发流（Prompt 工程核心区）：**
    - **Few-Shot 植入：** 让 Claude 提取过往爆款视频的行文结构，生成带有强烈“人设语气”和“画面感”的系统提示词。
    - **输出要求：** 严格要求模型输出 JSON 格式（包含 `title`, `spoken_text`, `camera_shots`, `subtitles`），方便下游解析。

### 节点 5：【数据清洗与排版】 (Code Node)
+ **功能描述：** Coze 内置 Python 环境，将大模型输出的 JSON 重新组装成易于市场部阅读的 Markdown 表格或文本块。
+ **代码逻辑：** 简单的键值提取与字符串拼接。

### [cite_start]节点 6：【审核提交】 (End/Webhook Node) [cite: 48]
+ [cite_start]**功能描述：** 提交给市场总监审核，为一键发布做准备 [cite: 48]。
+ **实现路径：** * 调用飞书/钉钉的自定义机器人 Webhook。将编排好的脚本一键推送到“开卓市场部内容审核群”。
    - **最终输出 (Output)：** 返回一条成功推送的通知给到触发工作流的用户。
