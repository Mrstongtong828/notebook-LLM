# 精密设备 AI 自动化运营项目执行任务书 (V3.0)
## 一、 团队架构与组织形式 (7人制)
+ **项目负责人 (1人 - 你)**：负责整体进度把控、核心算法逻辑终审及跨组协调。
+ **技术组 (2人)**：负责 OpenClaw 底层部署、Claude 3.5 Sonnet 提示词优化及 n8n 自动化流开发。
+ **运营组 (2人)**：负责闲鱼矩阵号维护、RPA 客户端实操及高价值订单的人工干预谈判。
+ **数据与知识组 (2人)**：负责 PDF 技术文档清洗、RAG 向量库构建及全网设备价格基准库维护。

---

## 二、 角色职责精细化清单 (Work Details)
### 1. 技术组 (Technical Duo)
+ **子任务 A**：完成 Docker 环境下 OpenClaw 的容器化集群部署，配置代理 IP 负载均衡。
+ **子任务 B**：针对 Claude 3.5 Sonnet 开发“深度设备扫描”Prompt，支持对暗病、铭牌参数的精准抓取。
+ **子任务 C**：开发 n8n 工作流，打通从“发现线索”到“获利计算”再到“飞书提醒”的全链路。

### 2. 运营组 (Operations Duo)
+ **子任务 A**：管理 6-10 个闲鱼矩阵账号，执行“养号计划”，负责日常账号活跃与反封控自检。
+ **子任务 B**：监控前端 RPA 响应状态，确保 AI 自动回复逻辑闭环。
+ **子任务 C**：执行“砍价指令”，在 AI 触达价格底线时接管会话，完成最后的商务闭环及发货物流。

### 3. 数据与知识组 (Knowledge Duo)
+ **子任务 A**：搜集并清洗实验室内所有精密仪器的 PDF 说明书，利用 Sonnet 提取核心技术指标。
+ **子任务 B**：构建并维护本地向量数据库，确保存储的专业知识能够精准支持 RAG 检索。
+ **子任务 C**：每日更新“市场价格雷达”，为技术组的评分系统提供最新的价格参考阈值。

## 四、 协作机制与风控
1. **每日晚会**：各组同步 DDL 达成情况，负责人协调技术与运营的配合偏差。
2. **风险隔离**：运营组负责的矩阵号与技术组的爬虫号物理隔离，防止 IP 连坐。
3. **决策权重**：500元以下配件由运营组直接决策；3000元以上设备需技术评分+负责人复核+导师终审。
